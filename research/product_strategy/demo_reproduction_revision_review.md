## Review Question

Did the revision resolve the external reproduction package hash drift without changing Domain 3 semantics, fixtures, adapters, producers, schemas, formal core, or product behavior?

## Findings

### F1 - Provenance Resolved

Severity: resolved

The three stale hashes came from generated Domain 3 conformance artifacts. The artifacts included volatile pytest output tails, specifically runtime-dependent text. The harness also wrote generated artifacts with CRLF working-tree bytes on Windows while the repository normalizes text to LF. This made the package manifest stale after a fresh conformance run even though the Domain 3 semantics were unchanged.

### F2 - Tooling Nondeterminism Removed

Severity: resolved

The Domain 3 conformance harness now records deterministic command summaries:

```text
stdout_summary
stderr_summary
```

instead of raw stdout/stderr tails. Runtime durations are normalized to:

```text
<duration>
```

A focused regression test was added to prevent reintroducing runtime-dependent command summaries. The harness also writes generated conformance artifacts with LF line endings before package hashes are computed.
